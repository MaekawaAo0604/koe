## 0. テンプレートの使い方

### 変数記法

テンプレート内の `[〇〇]` は送信時に必ず置き換えてください。変数一覧はセクション4を参照。

### 禁止事項

- **コピペ送信は禁止**。テンプレートはあくまで骨格
- 最低限のパーソナライズ必須要件:
  1. 相手の名前または @ハンドル を冒頭に入れる
  2. 相手の具体的な活動（制作物、発信内容）に触れる
  3. Koeの説明は最後の2〜3文に留め、冒頭に入れない

### DM送信前チェックリスト

- [ ] 相手の直近10投稿を読んだか？
- [ ] 1〜3日前に相手の投稿にリプライ/いいねをしたか？
- [ ] `[変数]` を全て具体的な内容に置き換えたか？
- [ ] DM全体が150文字以内に収まっているか？（初回DMは短く）

### 1-4. フォローアップ DM（初回返信なし時）

```
[相手の名前]さん

以前Koeのご連絡をさせていただいた[自分の名前]です。

```

### 1-5. 登録後フォローアップ DM

> **タイミング**: 登録から **48時間後** に送る。埋め込みまで完了してもらうためのオンボーディング支援。

```
[相手の名前]さん

Koeに登録いただきありがとうございます!

もうウィジェットのサイトへの埋め込みはできましたか?
もし詰まっているところがあれば教えてください、直接サポートします。

よくある詰まりポイント:
・scriptタグの設置場所 → </body>の直前に入れてください
・収集フォームURL → 既存クライアントにメッセージで送るのが一番早いです

何か分からないことがあればお気軽にどうぞ!
```

```

お客様の声・レビューを自動収集してサイトに埋め込めるツール
「Koe」をリリースしました。

- フォームURLを共有 → お客様の声が自動で集まる
- scriptタグ1行でサイトに埋め込み
- 無料から使えます

[KoeのURL]

フリーランスの方やポートフォリオをお持ちの方にぜひ試してほしいです。
感想・フィードバックいただけると嬉しいです!
```

### 3-2. 技術系Discord/Slack向け（体験談型）

> **使用場所**: #times チャンネルや雑談チャンネル。宣伝色を出さない。

```
個人でSaaSを作っているのですが、ユーザーのお客様の声（レビュー）を
LPに表示したくて、自分で「Koe」というツールを作りました。

scriptタグ1行でウィジェット表示、収集フォーム込みという構成です。
Shadow DOMで実装したので、既存サイトのCSSと干渉しません。

個人開発者の方で同じ課題がある方がいれば使ってみてください。
フィードバックも歓迎です → [KoeのURL]
```

### 3-3. フリーランス向けDiscord/Slack（課題共感型）

> **使用場所**: フリーランス向けコミュニティの雑談チャンネル。質問から入る。

```

自分はGoogleフォームで集めてHTML手動更新していたんですが、
面倒すぎて自動化するツールを作りました。

フォームURL共有 → 声が集まる → scriptタグ1行でサイトに自動表示
という流れです。

もし同じ課題を感じていたら試してみてほしいです → [KoeのURL]
無料から使えます。フィードバックも大歓迎です。
```

## 4. カスタマイズガイド（置き換え変数一覧）

| 変数 | 内容 | 例 |
|------|------|-----|
| `[相手の名前]` | X の表示名 or @ハンドル | 田中さん / @tanaka_design |
| `[相手の最近の制作実績投稿]` | 直近の投稿で印象に残ったもの | 「先週のECサイト制作事例のツイート」 |
| `[相手の投稿内容]` | 言及する投稿の具体的内容 | 「フリーランス3年目の振り返り」 |
| `[具体的な感想を1文]` | その投稿への自分の感想 | 「デザインの配色がすごく参考になりました」 |
| `[具体的な共感ポイント]` | 共感できるポイント | 「自分もクライアント獲得で同じ経験をしました」 |
| `[相手のサービスLP/ポートフォリオ]` | 相手のサイト名 | 「tanaka-design.comのポートフォリオ」 |
| `[良いと思った点を1文]` | サイトの具体的な良い点 | 「実績紹介の見せ方がとても分かりやすいです」 |
| `[相手の発信テーマ]` | 相手が主に発信している内容 | 「キャリアコーチング」 |
| `[相手のビジネスへの効果]` | Koeがその人のビジネスにどう効くか | 「新規クライアントの獲得に繋がりそう」 |
| `[相手のサービス名]` | 相手が提供しているサービス | 「TaskFlow」 |
| `[相手の活動]` | 相手の活動を簡潔に | 「Web制作のお仕事」 |
| `[自分の名前]` | 自分の表示名 | 前川 |
| `[KoeのURL]` | サービスのURL | （実際のURL） |
| `[相手のサイト名]` | 相手のサイト名称 | 「tanaka-design.com」 |
| `[具体的に良いと思った点]` | サイト閲覧後の感想 | 「制作実績の見せ方がとても分かりやすかったです」 |
| `[相手のサービス名/発信内容]` | メールでの相手への言及 | 「noteでのキャリアコーチング記事」 |
