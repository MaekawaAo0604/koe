# Koe - 営業戦略（フリーランス向けアウトリーチ）

> 本文書は [growth-strategy.md](growth-strategy.md) の Growth Channel 4（フリーランス特化の初期獲得）を「どうやるか」レベルに落とし込んだ実行文書です。
> [ROADMAP.md](../ROADMAP.md) Phase 1（Week 2-4）のスケジュールに合わせて設計しています。

## 前提

- 実行者: 1人（工数管理を最優先）
- 期間: Month 1（4週間）
- 言語方針: 「テスティモニアル」は使わない。「お客様の声」「レビュー」「感想」で統一
- 目標: 無料ユーザー 30〜50人のうち、アウトリーチ起点で 10〜15人を獲得

## 1. ターゲットペルソナ

### ペルソナA: Web制作フリーランス（最優先）

| 項目 | 内容 |
|------|------|
| 属性 | ポートフォリオサイト or 自社LPを持っている。既存クライアントが複数いる。Xでフォロワー200〜2,000人程度 |
| 課題 | クライアントから「実績を見せて」と言われるが、声を集める仕組みがない。Googleフォームで集めても表示が手動でHTML編集が必要 |
| Koeのメリット | フォームURLを送るだけで声が集まる。scriptタグ1行でポートフォリオに自動反映。将来的にアフィリエイト収入も狙える |
| 優先度 | **高** — 最もニーズが明確で、サイトを持っている確率が高い |

### ペルソナB: コーチ・コンサルタント

| 項目 | 内容 |
|------|------|
| 属性 | note / X で情報発信している。月5〜10万円以上を個人から受け取っている。メルマガ or LINEリスト保有 |
| 課題 | 高単価な仕事の受注にはクライアントの声が必要だが、インタビュー記事作成は時間がかかる。声をもらっても活用できていない |
| Koeのメリット | セッション後にフォームURLを送れば声が自動で集まる。Wall of Loveページが即座に生成される。LPへの埋め込みで成約率改善 |
| 優先度 | **中** — ニーズはあるが、ツール導入への抵抗が技術者より高い可能性あり |

### ペルソナC: 個人開発者・インディーハッカー

| 項目 | 内容 |
|------|------|
| 属性 | Zenn / Qiita に記事を書く。自作サービスのLPを持っている。#個人開発 ハッシュタグ利用 |
| 課題 | 作ったサービスのユーザーの声を集めたい。Xで言及されても一覧化できない |
| Koeのメリット | 自作サービスのユーザーの声を自動収集。技術者なので1行scriptタグの概念を即理解。Build in Publicとの親和性が高い |
| 優先度 | **中** — BIP発信との相乗効果で乗数がかかる可能性あり |

## 3. チャネル別アプローチ

### チャネル1: X DM（最重要）

**基本ルール**
- DMを送る前に必ず相手の投稿に「いいね」「返信」を **1〜3日以上** 行い、顔見知り感を作る
- DM1通あたりの作成時間は **5〜10分** かける（相手のアカウントを読んでパーソナライズ）
- **1日最大5通、週25通を上限** とする（負荷管理 + スパム回避）

**送ってよい条件**
- 相手が「お客様の声」「レビュー集め」「ポートフォリオ更新」に関連した投稿をしている
- 相手がサイトやLPを持っていることが確認できる
- フォロワー200〜5,000人程度（返信確率が高い帯域）

**送るべきでない条件**
- 最終投稿が2週間以上前のアカウント
- フォロワー5万人以上のアカウント（返信確率が低い）
- プロフィールに「DM不可」「営業お断り」の記載

**テンプレート**: → [outreach-templates.md](outreach-templates.md) セクション1 を参照

### チャネル2: コミュニティ投稿（Discord/Slack）

**投稿方針**
- 宣伝チャンネルがある場合のみ告知投稿
- それ以外は自分のKoe利用体験を #times や雑談チャンネルで共有する形（押し付けない）
- 質問チャンネルで「お客様の声どう集めてますか?」への回答として自然に紹介

**テンプレート**: → [outreach-templates.md](outreach-templates.md) セクション3 を参照

### チャネル3: メール（サイトにメアドがある場合）

**対象**: Web制作会社の個人ページ、コーチ/コンサルのLPにメアドが掲載されているケース

**ルール**
- **週5通以下** に限定
- サイトを実際に見て、具体的なページ名を本文に入れる
- 件名はKoeの宣伝にせず、相手の課題ベースにする

**テンプレート**: → [outreach-templates.md](outreach-templates.md) セクション2 を参照

## 4. パーソナライズの原則

### スパムになるパターン（やってはいけない）

- 相手の投稿を一切読まずに同じ文面を送る
- 「よろしければお使いください」で完結するDM
- 「初めまして。私は○○を作りました。」から始まるDM
- URLだけ送る
- 1日に10通以上の同一文面を送る

### 3段階プロセス

**段階1 — 観察（DM送信前）**
- 相手の直近10投稿を確認
- 「お客様の声に困っている」「ポートフォリオを更新した」「クライアント対応」に関する投稿がないか確認
- 相手が何を売っているか（Web制作、コーチング、SaaS等）を把握

**段階2 — 関係構築（DM前1〜3日）**
- 相手の投稿に**具体的なリプライ**を1〜2回送る（いいねだけではなく文章で返す）
- 相手が反応したらDMを送る機会と判断

**段階3 — DM送信**
- 書き出しは相手の具体的な投稿内容への言及
- Koeの説明は1〜2文で完結させる
- 「使ってみませんか」ではなく「試してみる価値があるか確認してほしい」の姿勢
- 最後に「合わなければスルーで大丈夫です」を入れる

## 5. KPI

### 週次KPI

| 指標 | 週次目標 | 計測方法 |
|------|----------|----------|
| ターゲット候補の観察数 | 20〜30人 | Xリストの更新数 |
| 相手への返信/いいね数 | 15〜20件 | 手動カウント |
| DM送信数 | 10〜15通 | スプレッドシートで管理 |
| DM返信率 | 目標 **20%以上** | 返信数 / 送信数 |
| 返信→登録の転換率 | 目標 **30%以上** | 登録ユーザーの流入元確認 |
| コミュニティ投稿数 | 1〜2本/週 | 手動カウント |

### Month 1 数値目標（ROADMAP.md Phase 1 と整合）

| 指標 | 目標 |
|------|------|
| 無料ユーザー（全体） | 30〜50人 |
| うちアウトリーチ起点 | 10〜15人 |
| 有料転換 | 0〜3人 |

## 6. 週次アクションプラン

### Week 1: 準備フェーズ

- [ ] ペルソナA・B・CのXリストを作成し、各10〜20人を追加
- [ ] 各リストの投稿を読んでパーソナライズ用メモ作成
- [ ] [outreach-templates.md](outreach-templates.md) のDMテンプレートをセグメント別にカスタマイズ
- [ ] コミュニティの場所を特定（Discord招待リンク取得など）
- [ ] 知り合いのフリーランス1人への初回アプローチ（ROADMAP.md Step 1）
- [ ] 管理用スプレッドシート作成

### Week 2: 関係構築

- [ ] 知り合いフリーランス経由で紹介2〜3人の獲得を目指す
- [ ] Xでペルソナの投稿への返信を開始（1日3〜5件）
- [ ] DMは**まだ送らない**（関係構築期間）
- [ ] コミュニティの宣伝チャンネルに1本投稿
- [ ] 自分のサービスでドッグフーディング（Koe自体のお客様の声を集める）

### Week 3: DM送信開始

- [ ] Week 2で返信を交わした相手にDMを送り始める（週10〜15通）
- [ ] 返信が来たユーザーとの会話を丁寧に続ける
- [ ] Zenn/Qiita 技術記事を投稿してペルソナCへのリーチを開始
- [ ] KPIの計測を開始（スプレッドシートで管理）

### Week 4: 計測・改善

- [ ] Week 3のDM返信率を集計
- [ ] 返信率が低いセグメントのテンプレートを見直す
- [ ] 有料転換が1件でも出た場合はその人の属性を深掘りしてターゲットを絞り直す
- [ ] 翌月のアクションプランを調整

## 7. 失敗パターンと対処法

| 失敗パターン | 原因 | 対処法 |
|-------------|------|--------|
| DMを大量送信してアカウント制限される | 1日20通以上の送信 | 1日5通の上限を厳守 |
| 返信率が5%未満 | パーソナライズ不足 | テンプレート見直し + 関係構築フェーズを延長 |
| 登録はするが埋め込みまで至らない | オンボーディング問題 | 登録後48時間以内にフォローDMで使い方を案内 |
| コミュニティで「宣伝」と思われる | チャンネルルール無視 | 宣伝チャンネル以外では体験談として話す |
| 返信が来ても会話が続かない | クロージングが弱い | 具体的な次のアクション（「5分で試せます」）を提示 |
| 1人で回しきれない | 週25通 + 関係構築で工数オーバー | DM数を週10通に絞り、ペルソナAに集中 |
